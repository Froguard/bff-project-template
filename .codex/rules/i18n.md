# 国际化规则

## 语言包与文案复用

- 新增或修改多语言 key 时，必须确保 `resources` 下每个语言资源文件具有相同的 key 结构，并执行 `pnpm i18ncheck` 验证。
- 多语言 key 应按可复用的业务语义组织，避免随页面或组件无序扩张：
  - 新增文案前必须先检索并优先复用已有的同语义 key。
  - 按钮、状态、语言、应用、请求、主题等通用文案优先归入 `common`、`actions`、`language`、`app`、`request`、`theme` 等公共顶层域，而不是具体页面或组件域。
  - 当同一语义的文案在两个及以上位置使用时，应将其提升到合适的公共顶层域并统一替换原有重复 key。
  - 仅在语义和使用上下文一致时复用，禁止为了减少 key 而混用含义不同的文案。
